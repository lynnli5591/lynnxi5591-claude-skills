# 案例复盘：企业薪资管理系统原型

## 项目背景

**输入**：业务场景截图（薪资数据收集系统的现有流程）  
**输出**：完整可交互 HTML 原型（管理员端 + HR 端，16 个功能页面）  
**迭代轮数**：30+ 轮对话，从架构到细节逐步完善

---

## 产品架构（A-F 六层）

| 层 | 名称 | 核心职责 |
|---|---|---|
| A | 上游数据层 | 门禁/处罚/绩效系统 API 接入，增量同步 & 全量回刷 |
| B | 预算管控层 | 集团级预算总额设定、实时余额追踪、超额审批流 |
| C | 薪资规则层 | 薪资档次划分、薪资项目档案、人员档次匹配 |
| D | 员工申请层 | 资格准入判断、申请发起、状态追踪 |
| E | 校验&审核层 | 系统自动校验（管理员处理）→ 放行 → HR 复核 |
| F | 核算发放层 | 核算明细生成、一键发放 |

## 角色设计

### 管理员端

**核心职责**：系统配置、规则维护、数据质量管控

| 功能模块 | 说明 |
|---|---|
| 薪资档次划分 | 档次 CRUD，支持草稿/审批流，关联薪资项目 |
| 人员档次匹配 | 自动匹配 + 手动覆盖，AI 推荐边界人员 |
| 薪资项目档案 | 基础档案，与档次多对多关联 |
| 规则变更记录 | 变更历史时间轴，待审批变更卡片 |
| 校验规则库 | 5 类规则，3 种处理级别（阻断/警告/提示） |
| 校验异常处理 | 按规则类型动态展示处理方式，处理完放行申请 |
| 预算设定 | 集团级预算、部门分配（按月×薪资类型） |

### HR 专员端

**核心职责**：业务审批、预算追踪、核算执行

| 功能模块 | 说明 |
|---|---|
| HR 工作台 | 流程进度总览、AI 分析、预算燃尽图 |
| 申请复核 | 只看已通过校验的干净数据，通过/驳回 |
| 预算追踪 | 实时余额、超额审批记录 |
| 核算明细 | 一键生成、导出 |
| 上游源数据 | 办公地点/处罚/绩效数据查看，支持同步 |

## 关键设计决策

### 决策1：校验异常归管理员，不归 HR

**背景**：最初将数据校验放在 HR 端

**问题**：HR 需要同时处理数据质量问题（哪个系统数据错了）和业务审批（这个申请合不合理），职责混乱

**结论**：
- 校验异常由管理员/数据专员处理，处理完才放行
- HR 看到的申请列表是已通过校验的干净数据
- HR 工作台显示"校验异常（管理员处理中）"，只读，不可操作

### 决策2：校验级别 × 异常类型 = 动态处理方式

**背景**：弹窗里三个处理方式写死，对所有异常通用

**问题**：「按档次上限重新计算」对「人员不在资格名单」这类异常根本无法使用

**结论**：按 `type × level` 配置处理方式：

| 异常类型 | 级别 | 处理方式 |
|---|---|---|
| 金额类 | 阻断 | 修正为上限 / 手动覆盖 / 升级特批 |
| 资格类 | 阻断 | 以人事系统为准 / 以系统为准（豁免）/ 驳回 |
| 预算类 | 警告 | 知悉记录 / 申请超额审批 |
| 时效类 | 警告 | 豁免本次 / 驳回 |
| 数据类 | 警告 | 以系统为准 / 以档案为准 / 标记复核 |

### 决策3：预算设定归管理员端

**背景**：预算设定放在 HR 端「预算追踪」的第二个 Tab

**问题**：预算总额设定是高权限操作，HR 不应该有配置权

**结论**：
- 管理员侧边栏新增「预算管控」分区，含「预算设定」页面
- HR 端「预算追踪」只保留「实时余额追踪」和「超额审批」两个 Tab
- 原「预算总额设定」Tab 改为引导提示，跳转管理员端

## 原型技术实现要点

### 导航索引 Bug（出现过两次）

**现象**：菜单高亮跑偏，点击A高亮B

**原因**：`adminSidebarMap` 的索引值和 `.sidebar-item` 实际 DOM 顺序不一致，将 `.sidebar-section`（分区标题）和 `.divider` 算进去了

**修复**：只数 `.sidebar-item` 元素，手动核对 DOM 顺序

### 模板字符串语法错误

**现象**：Console 报 `Missing } in template expression`

**原因**：模板字符串里的三元表达式末尾多了分号
```javascript
// 错误
`<td style="color:${r.pct > 85 ? 'red' : 'black';}">` 
// 正确
`<td style="color:${r.pct > 85 ? 'red' : 'black'}">`
```

**预防**：每次改动后运行 `node --check` 验证

### 抽屉索引传递

档次详情抽屉和编辑抽屉同时存在时，需要一个共享变量：
```javascript
let currentGradeIndex = 0; // 详情抽屉用
let gradeEditIndex = -1;   // 编辑抽屉用
```
详情抽屉的「编辑」按钮要用 `currentGradeIndex` 打开编辑抽屉。
